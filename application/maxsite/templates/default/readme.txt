ЧЕКЛИСТ СОЗДАНИЯ НОВОГО ШАБЛОНА
...............................

	1. Создайте новый каталог шаблона.

	2. Скопируйте в него содержимое _create_a_new_template.

	3. Переходим в каталог нового шаблона.

	4. Откроем info.php и укажем данные шаблона.

	5. Перейдем в админ-панель: Настройки основные - Шаблон сайта и переключимся
	   на новый шаблон. 
	   
	   Примечание. Если вы используете плагин Theme switch, то в его настройках
	   необходимо выбрать новый шаблон. После этого переключиться на новый шаблон 
	   можно в виджете Theme switch на сайте. 

	   Отключите все профили оформления в Настройке шаблона.

	6. Согласно дизайна подберите наиболее подходящие компоненты шапки и подвала 
	   сайта. Если таких компонентов нет, то создайте файлы-заглушки для будущих
	   компонентов.
	   
	   После этого в Настройках шаблона - Настройка блоков выберите компоненты.
	
	7. Задайте базовую структуру сайта. 
	   
	   Примечание. Посмотреть стили можно в default/css/style-all.css. В файле 
	   стили сгруппированы по секциям. Файл default/css/var_style.css содержит
	   описание структуры, а также некоторых элементов, которые могут меняться в
	   других шаблонах. Именно поэтому они вынесены в отдельный файл.
	   
	   Важно! Файл style-all-mini.css и/или style-all.css НЕ НУЖНО копировать
	   в свой шаблон. Он автоматически подключится из default-шаблона.
	   
	   Если структура шаблона совпадает с default, то используем только 
	   css/add_style.css.
	   
	   Если структура отличается от default, то копируем из default-шаблона
	   css/var_style.css в свой шаблон и меняем структуру в css/var_style.css
	   своего шаблона.
	   
	   При использовании своего var_style.css можно не использовать 
	   add_style.css.

	8. Если на сайте используется несколько сайдбаров. 
	
		8.1 Необходимо задать сайдбары в custom/my_functions.php
	   
			mso_register_sidebar('2', t('Второй сайдбар', 'templates'));
	
	        Этот код подключит настройку второго сайдбара в админ-панели.
	   
		8.2 Для вывода нескольких сайбаров используется файл custom/sidebars.php.
			Обратите внимание, что этот файл полностью отвечает за вывод сайдбаров,
			включая php-код и HTML-разметку с css-классами.
	   
			Вы можете переопределить css-классы: например задать общий 
			div.sidebars, убрать div.sidebar, оставив только div.sidebar1 и
			div.sidebar2:
	       
				echo '<div class="sidebars"><div class="sidebars-wrap">';

					echo '<div class="sidebar1"><div class="sidebar1-wrap">';
						mso_show_sidebar('1');
					echo '</div></div>';

					echo '<div class="sidebar2"><div class="sidebar2-wrap">';
						mso_show_sidebar('2');
					echo '</div></div>';

				echo '</div></div>';
	       
			Данный подход хорош тем, что не потребуется переопределять существующие
			стили из default-шаблона.
			
		8.1 Настройте положения сайдбаров в var_style.css или add_style.css.
	   
	   
	9. Задайте положение и фон подвала в var_style.css или add_style.css.
	
	10. Задайте основные стили
		- шрифт -> body
		- стиль ссылок -> а
		--- стиль ссылок в сайдбара/виджетах -> div.sidebar(s) a
		- стиль заголовков -> h1, h2, h3...
		
		Для контроля создайте на сайте произвольную страницу, где разместите
		bb-код [text-demo]. Теперь на этой странице будет выводится тестовый 
		типовой набор стилей и html-тэгов. Добейтесь, отображения, соответствующее
		дизайну сайта.

	11. Задайте оформление виджетов в сайдбарах.
		- Заголовки виджетов -> div.widget_header
		- Списки в виджетах ->
			ul.is_link li - элемент первого уровня
			ul.is_link li li - второй уровень

	12. Указываем (если нужно) для основного контента (main и sidebar) фоновую 
	    картинку (div.main)
		
	13. Указываем (если нужно) Фоновую картинку браузера (body или div.all). 
	    Если фоновая картинка зависит от шапки (компонентов), то фон браузера
	    выставляем уже после настройки компонентов шапки.
	    
	14. Если вы хотите использовать инфо-блок записи, как в default, то 
	    скопируйте каталог type_foreach в свой шаблон.
	
	15. Настраиваем вывод 
		- Оставьте комментарий! - div.leave_a_comment
		- Заголовок рубрики h1.category
		- Еще записи по теме - div.page_other_pages_header
		- Комментариев - div.page_comments_count
	
	16. Настраивам, при необходимости форму комментирования. См. 
	    секцию COMMENT FORM.
	
	17. Создаем компоненты шапки. Вначале следует настроить те элементы, которые
	    не требуют переделки php/html-кода. Оформление настраивается 
	    в add_style.css. Типичный пример - меню (components/menu.php).
		
		После можно настроить компоненты, требующие создания/переделки php-кода.
		
		В завершении беремся за те компоненты, которые используют отдельные 
		опции и js-код.
		
	18. Аналогично поступаем с компонентами подвала.
	
	19. Настраиваем вывод, если требуется с помощью custom-файлов.
	
	20. Удаляеем из шаблона неиспользуемые или неподдерживаемые компоненты и
		профили оформления.
